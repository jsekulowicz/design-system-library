# Page Shell

Application frame that arranges a sticky header, a scrollable aside, a main content region, and an optional footer. Header chrome stretches to the viewport edges; the _content column_ (header inner content and the aside + main row) can be capped with `--ds-page-shell-max-width` and centred in the viewport, so brand, sidenav, and main all align vertically on wide screens when a cap is set.

## Anatomy

- **header** — sticky top bar with full-bleed border + backdrop blur. Inner content (`brand` slot left, `header-actions` slot right) follows the same `--ds-page-shell-max-width` as the body.
- **aside** — side column; collapses below the header under the tablet breakpoint. When the slot is empty the column, hamburger toggle, and drawer are not rendered (single-section apps). Add `aside-toggle` to render a desktop-only control that cycles the start aside through full, compact, and hidden states.
- **main** — grows to fill remaining vertical space within the shell body. A direct `ds-scrollable-page` child automatically takes over scrolling and keeps its optional header outside the scroll region.
- **aside-end** — optional inline-end region. Add `aside-end-toggle` to render a desktop-only control that hides or shows it.
- **footer** — optional bare slot beneath the body. Slot `ds-footer` directly when the page needs footer chrome; when the slot is empty PageShell renders no footer markup.

```html
<ds-page-shell brand="Brand" aside-toggle>
  <div slot="header-actions">...</div>
  <ds-sidenav slot="aside">...</ds-sidenav>
  <article><!-- page content --></article>
  <ds-footer slot="footer">...</ds-footer>
</ds-page-shell>
```

For a non-scrolling page heading with a scrollbar at the main region's edge, compose the scrollable page template directly inside PageShell:

```html
<ds-page-shell brand="Brand">
  <ds-sidenav slot="aside">...</ds-sidenav>
  <ds-scrollable-page>
    <header slot="header">
      <h1>Activity</h1>
      <p>Recent workspace events.</p>
    </header>
    <section>...</section>
  </ds-scrollable-page>
</ds-page-shell>
```

### Content cap

The shell exposes `--ds-page-shell-max-width` (default `none`) so consumers can opt into a centred content column without forking the component:

```css
ds-page-shell {
  --ds-page-shell-max-width: 90rem; /* cap wide application frames */
}
```

The constrained PageShell story shows the capped layout. The other PageShell and SettingsPage stories use the default fluid width.

The collapsible-asides story shows the mirrored start/end controls together. Desktop asides hide
native scrollbars and use the shared top/bottom content fade when more content is available.

      <h3>Do</h3>
      <ul>
        <li>Slot a <code>ds-sidenav</code> into <code>aside</code> — it handles collapsed mode, icons-only state, and keyboard navigation.</li>
        <li>Use <code>aside-toggle</code> on <code>ds-page-shell</code> when users should be able to compact or hide the start aside.</li>
        <li>Use <code>aside-end-toggle</code> when users should be able to hide a secondary inline-end aside.</li>
        <li>Use a direct <code>ds-scrollable-page</code> child when a non-scrolling page header and edge-aligned content scrollbar are needed.</li>
        <li>Slot a <code>ds-footer</code> into <code>footer</code> for a consistent bottom strip across pages.</li>
        <li>Constrain main content width (e.g. <code>max-width: 68ch</code> or <code>72rem</code>) inside the default slot for comfortable line lengths.</li>
      </ul>
      <h3>Don't</h3>
      <ul>
        <li>Don't place unbounded content in the <code>aside</code> slot — the aside is independently scrollable; keep its content concise.</li>
        <li>Don't nest another <code>ds-page-shell</code>; one frame per page is the intended use.</li>
        <li>Don't rely on the <code>brand</code> attribute for rich markup — use the <code>brand</code> slot instead so you can pass a logo element.</li>
      </ul>
      <h3>Accessibility</h3>
      <ul>
        <li>Each region maps to a native landmark: <code>&lt;header&gt;</code>, <code>&lt;aside&gt;</code>, <code>&lt;main&gt;</code>, <code>&lt;footer&gt;</code> — screen readers announce them without extra <code>role</code> attributes.</li>
        <li>The sticky header uses a semi-transparent backdrop; verify contrast ratios when the page has coloured hero content scrolling beneath it.</li>
        <li>At narrow viewports the aside collapses into a stacked panel above <code>main</code> — ensure sidenav items are reachable in that order.</li>
      </ul>
