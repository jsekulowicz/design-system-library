# Checkbox

A single selectable option. Supports `indeterminate` for tri-state parent/child relationships (e.g. "select all").

## Anatomy

A native `<input type="checkbox">` in shadow DOM with a custom SVG indicator. Pair with `ds-checkbox-group` to collect values under one fieldset.

    <ds-tab-panel value="guidelines">
      <h3>Do</h3>
      <ul>
        <li>Use for settings and multi-select lists where each option is independent.</li>
        <li>Use <code>indeterminate</code> on a parent that aggregates a partial child selection.</li>
        <li>Prefer a descriptive label over generic "Accept" — "Email me weekly summaries" beats "Subscribe".</li>
      </ul>
      <h3>Don't</h3>
      <ul>
        <li>Don't use a checkbox for mutually exclusive options — reach for <code>ds-radio</code>.</li>
        <li>Don't toggle global settings with a checkbox; use a dedicated switch component.</li>
        <li>Don't hide the label visually — it is the click target and the a11y name.</li>
      </ul>
      <h3>Accessibility</h3>
      <ul>
        <li>Clicking the label toggles the box via native <code>&lt;label&gt;</code> association.</li>
        <li>Keyboard: <code>Space</code> toggles; focus ring uses <code>:focus-visible</code>.</li>
        <li><code>indeterminate</code> is reflected via the <code>aria-checked="mixed"</code> state.</li>
      </ul>
    </ds-tab-panel>
