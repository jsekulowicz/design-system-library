# Icon

Renders SVGs registered via `registerIcon(name, svgMarkup)`. Icons are sourced from [Heroicons](https://heroicons.com) (MIT).

## Anatomy

Each icon module self-registers on import, so only the ones you import end up in your bundle. The component resolves the registered SVG by `name` at render time.

      <h3>Using icons in your project</h3>
      <p>First define the web component once. This registers <code>&lt;ds-icon&gt;</code>, but it does not register any icon artwork:</p>
      <pre><code>import '@jsekulowicz/ds-components/icon/define';</code></pre>
      <p>Every Heroicons 24 outline icon is available as a kebab-case side-effect module. Import only the icons you use; each module self-registers that icon name:</p>
      <pre><code>import '@jsekulowicz/ds-components/icon/academic-cap';{'\n'}import '@jsekulowicz/ds-components/icon/arrow-right';</code></pre>
      <pre><code>&lt;ds-icon name="academic-cap"&gt;&lt;/ds-icon&gt;</code></pre>
      <p>For prototypes, demos, or documentation, register the full Heroicons set at once:</p>
      <pre><code>import '@jsekulowicz/ds-components/icon/all';</code></pre>
      <p>To inspect available names without registering SVGs, import the generated name list:</p>
      <pre><code>import {'{'} heroiconNames, type HeroiconName {'}'} from '@jsekulowicz/ds-components/icon/names';</code></pre>
      <p>To register a custom icon or a different Heroicons variant, use the registry API:</p>
      <pre><code>import {'{'} registerIcon {'}'} from '@jsekulowicz/ds-components/icon';{'\n'}registerIcon('my-icon', '&lt;svg viewBox="0 0 16 16" fill="currentColor"&gt;...&lt;/svg&gt;');</code></pre>

      <h3>Tree shaking</h3>
      <ul>
        <li>Individual icon imports are tree-shake friendly: <code>import '@jsekulowicz/ds-components/icon/arrow-right'</code> only registers that icon module.</li>
        <li>Unimported icon modules are not included just because a template contains <code>&lt;ds-icon name="..."&gt;</code>. The <code>name</code> property is only a runtime lookup key.</li>
        <li><code>@jsekulowicz/ds-components/icon/all</code> intentionally imports every generated icon, so it is not tree-shaken down to only the names you render.</li>
        <li><code>@jsekulowicz/ds-components/icon/names</code> exports the string list and type only; it does not register SVG artwork.</li>
      </ul>

      <h3>Do</h3>
      <ul>
        <li>Pair icons with a text label when the meaning isn't universal.</li>
        <li>Use <code>label</code> to provide an accessible name for standalone icons.</li>
        <li>Match the icon size to the surrounding text via <code>size</code> or <code>font-size</code>.</li>
      </ul>
      <h3>Don't</h3>
      <ul>
        <li>Don't rely on icons alone to communicate critical status.</li>
        <li>Don't mix outline and solid variants in the same surface.</li>
        <li>Don't register icons inline per render — import the module once.</li>
      </ul>
      <h3>Accessibility</h3>
      <ul>
        <li>Decorative icons inherit <code>aria-hidden="true"</code> — screen readers skip them.</li>
        <li>Set <code>label</code> to promote the icon to an <code>img</code> with an accessible name.</li>
      </ul>
