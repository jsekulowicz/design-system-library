# Select

Native-feeling dropdown supporting single or multi-select. Options are provided declaratively via the `options` property.

## Anatomy

A custom trigger and a listbox popover rendered in shadow DOM. Supports `clearable`, `required`, `multiple`, and a `leading` slot for icons. Works as a form control via `ElementInternals`.

    <ds-tab-panel value="guidelines">
      <h3>Do</h3>
      <ul>
        <li>Use when options are known and modest in number (up to ~20). For longer lists, prefer <code>ds-searchable-select</code>.</li>
        <li>Provide a clear <code>placeholder</code> that describes the action — "Pick a discipline", not just "Select…".</li>
        <li>Show <code>error</code> messages close to the select, always below it.</li>
      </ul>
      <h3>Don't</h3>
      <ul>
        <li>Don't use for 2–3 mutually exclusive options — use <code>ds-radio-group</code> for faster scanning.</li>
        <li>Don't hide critical options behind <code>multiple</code> mode when single selection would do.</li>
        <li>Don't pre-select a destructive option as default.</li>
      </ul>
      <h3>Accessibility</h3>
      <ul>
        <li><code>role="combobox"</code> on the trigger, <code>role="listbox"</code> on the popover.</li>
        <li>Keyboard: <code>Space</code> / <code>Enter</code> opens; arrow keys move the active option; <code>Esc</code> closes.</li>
        <li>Focus is trapped inside the popover while open.</li>
      </ul>
