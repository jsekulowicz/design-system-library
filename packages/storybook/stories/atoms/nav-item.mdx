# NavItem

A single navigation link. Renders an `<a>` (or `<span>` when disabled) with optional leading icon and a `current` state that maps to `aria-current="page"` and a visible active style.

`ds-nav-item` is the link primitive used inside `ds-sidenav` (sidebar). The consumer owns the `current` state — the parent does not coordinate which item is active. Set `current` based on the URL.

## Anatomy

```html
<ds-nav-item href="/" current>
  <ds-icon slot="icon" name="home" size="lg"></ds-icon>
  Home
</ds-nav-item>
```

Items live in `ds-sidenav` as a flex column. When the closest ancestor is a `ds-sidenav[collapsed]`, the label is visually clipped — only the icon remains visible. The accessible name is preserved.

    <ds-tab-panel value="guidelines">
      <h3>Do</h3>
      <ul>
        <li>Set <code>current</code> on exactly one item per nav region, based on the active URL.</li>
        <li>Provide an <code>icon</code> slot when the item will appear inside a collapsible <code>ds-sidenav</code> — collapsed mode hides the label.</li>
        <li>Pair <code>target="_blank"</code> with <code>rel="noopener"</code> for external links.</li>
      </ul>
      <h3>Don't</h3>
      <ul>
        <li>Don't use <code>ds-nav-item</code> for inline document links — use <code>ds-link</code> for those.</li>
        <li>Don't disable items as a navigation gate; route the user to a "coming soon" page instead so they can read why.</li>
      </ul>
      <h3>Accessibility</h3>
      <ul>
        <li><code>current</code> reflects to <code>aria-current="page"</code> on the inner anchor.</li>
        <li><code>disabled</code> renders a non-interactive <code>&lt;span&gt;</code> with <code>aria-disabled="true"</code>.</li>
        <li>Inside a collapsed sidenav, the label remains in the accessibility tree (visually clipped, not <code>display:none</code>).</li>
      </ul>
