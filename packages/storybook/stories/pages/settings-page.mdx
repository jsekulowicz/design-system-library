# Settings Page

Full-page settings layout with a sticky section nav and scrollable form regions, composed inside `ds-page-shell`.

## Anatomy

`ds-settings-page` renders a hero header above a two-column grid. The left column is a sticky nav generated from the `sections` prop; the right column hosts the default slot. Wrap it in `ds-page-shell` to get the application chrome (sticky top bar, sidenav, footer).

```html
<ds-page-shell brand="Brand">
  <ds-sidenav slot="aside">
    <ds-nav-item href="/settings" current>
      <ds-icon slot="icon" name="cog-6-tooth"></ds-icon>
      Settings
    </ds-nav-item>
  </ds-sidenav>
  <ds-settings-page
    eyebrow="Workspace · Brand"
    heading="Settings"
    .sections=${[{ id: 'profile', label: 'Profile' }]}
  >
    <section id="profile">
      <ds-form header="Profile">
        <ds-text-field label="Display name"></ds-text-field>
        <ds-button slot="actions" variant="primary" size="sm">Save</ds-button>
      </ds-form>
    </section>
  </ds-settings-page>
  <ds-footer slot="footer">
    <span slot="start">© 2026 Brand</span>
  </ds-footer>
</ds-page-shell>
```

  <ds-tab-panel value="guidelines">
    <h3>Do</h3>
    <ul>
      <li>Match every entry in the <code>sections</code> prop to a <code>&lt;section id="..."&gt;</code> in the default slot — the nav links scroll to these targets.</li>
      <li>Use <code>ds-form</code> inside each section to get consistent layout, header, and actions slot alignment.</li>
      <li>Place <code>ds-settings-page</code> in <code>ds-page-shell</code>'s default slot — it fills the main content area and the page-shell provides the sticky header, sidenav, and footer.</li>
    </ul>
    <h3>Don't</h3>
    <ul>
      <li>Don't add <code>sections</code> entries without a matching <code>id</code> in the slot — the nav link will scroll nowhere.</li>
      <li>Don't pass rich markup through <code>eyebrow</code>, <code>heading</code>, or <code>description</code> — these are plain-text attributes; use CSS parts (<code>hero</code>, <code>heading</code>) for visual overrides.</li>
      <li>Don't nest another <code>ds-settings-page</code>; one settings layout per page is the intended pattern.</li>
    </ul>
    <h3>Accessibility</h3>
    <ul>
      <li>The sticky nav is a <code>&lt;nav aria-label="Settings sections"&gt;</code> landmark — screen readers announce it separately from the main content.</li>
      <li>Active nav links carry <code>aria-current="true"</code> (updated on click) so assistive technologies can identify the current section.</li>
      <li>On narrow viewports the nav collapses to a single column above the sections — ensure section headings remain visible and meaningful without the nav context.</li>
    </ul>
