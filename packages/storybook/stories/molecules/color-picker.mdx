# ColorPicker

Form-associated color picker for selecting a normalized Hex RGB value from preset swatches or a custom color.

## Anatomy

A field trigger opens a responsive card panel. Presets render as an accessible radiogroup, and custom color editing uses a native color input plus a Hex text input.

    <ds-tab-panel value="guidelines">
      <h3>Do</h3>
      <ul>
        <li>Use predefined <code>colors</code> for brand, theme, or status palettes.</li>
        <li>Provide human labels for swatches when color names are meaningful.</li>
        <li>Use <code>required</code> when a missing color would break the downstream workflow.</li>
      </ul>
      <h3>Don't</h3>
      <ul>
        <li>Don't use unsupported color formats in v1; values normalize to <code>#RRGGBB</code>.</li>
        <li>Don't rely on color alone — labels and selected state are exposed to assistive tech.</li>
        <li>Don't place the picker in containers that clip overflow unless the popover can remain visible.</li>
      </ul>
      <h3>Accessibility</h3>
      <ul>
        <li>The trigger exposes <code>aria-haspopup="dialog"</code> and reflects expanded/invalid state.</li>
        <li>Preset swatches use <code>role="radio"</code>, <code>aria-checked</code>, and roving keyboard focus.</li>
        <li>Keyboard: <code>Enter</code> / <code>Space</code> opens and selects, arrows move among swatches, <code>Escape</code> closes.</li>
      </ul>
