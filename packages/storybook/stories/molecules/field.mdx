# Field

A thin prop-forwarding wrapper. Pass `label`, `help`, `error`, and `optional` on `ds-field` and it sets them as attributes on the slotted control. The control (e.g. `ds-text-field`, `ds-select`) is responsible for rendering the label and subtext in its own shadow DOM.

## Anatomy

```html
<ds-field label="Display name" help="Shown on receipts.">
  <ds-text-field name="name"></ds-text-field>
</ds-field>
```

This is equivalent to writing:

```html
<ds-text-field label="Display name" description="Shown on receipts." name="name"></ds-text-field>
```

`ds-field` maps `help` → `description` on the control, and forwards `error` and `optional` verbatim.

    <ds-tab-panel value="guidelines">
      <h3>Do</h3>
      <ul>
        <li>Use <code>ds-field</code> when you want to set label/help/error in one place rather than on the control directly — it's especially useful inside form loops or when props are data-driven.</li>
        <li>Mark truly optional fields with <code>optional</code>; assume required by default and only label the exceptions.</li>
        <li>Put help copy in <code>help</code>, not in the label — labels should be short nouns.</li>
      </ul>
      <h3>Don't</h3>
      <ul>
        <li>Don't set <code>label</code> (or <code>description</code> / <code>error</code>) both on <code>ds-field</code> and directly on the slotted control — <code>ds-field</code> will overwrite what you set on the control.</li>
        <li>Don't replace inline errors with toast notifications — the message belongs next to the control.</li>
      </ul>
      <h3>Accessibility</h3>
      <ul>
        <li>The <code>label</code> prop is forwarded to the slotted control, which renders it as a <code>&lt;label&gt;</code> element in its own shadow DOM.</li>
        <li>Help and error text are forwarded as <code>description</code> and <code>error</code> attributes; each control wires them to the input via <code>aria-describedby</code>.</li>
      </ul>
